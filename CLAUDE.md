# goto

> `AGENTS.md` is a symlink to this file.

Two install surfaces for macOS project jumping:
**`goto`** (CLI picker) and **`Goto.app`** (menu bar UI + Finder Sync host with embedded extension).

## Tech stack

- **CLI** (`goto`): Node.js >= 20, zero dependencies, ESM (`"type": "module"`)
- **Native App** (`Goto.app`): Swift 5.8, SwiftUI, macOS 13+ вҖ” Xcode project (`macos/Goto.xcodeproj`), generated by Ruby script

## Key directories

| Path | Purpose |
|------|---------|
| `src/` | CLI source (commands, select, output) |
| `native/` | SPM package вҖ” shared native core plus legacy `GotoMenuBar` target/tests |
| `macos/Goto/` | Xcode вҖ” unified app host (menu bar UI + settings window) |
| `macos/GotoFinder/` | Xcode вҖ” Finder launch bridge used by `Goto.app` |
| `macos/GotoFinderSync/` | Xcode вҖ” Finder Sync extension (sandboxed) |
| `macos/Resources/` | Native app resources such as `Goto.icns` |
| `macos/Shared/` | IPC notification definitions shared by agent + extension |
| `shell/` | Shell integration (sourced by user's shell profile) |
| `scripts/` | Build & codegen scripts |
| `test/` | Node.js tests |
| `bin/` | CLI entry point (`goto.js`) |

## Build & test commands

```bash
node --test                          # 27 JS tests вҖ” must pass before changes
swift test -c debug                  # 38 Swift tests вҖ” must pass before changes
swift build -c release               # Build SPM targets
scripts/build-finder.sh              # Build Goto.app (.app + .appex)
scripts/install-finder.sh            # Install Goto.app to ~/Applications
scripts/generate_macos_project.rb    # Regenerate Xcode project
```

## Shared registry

- The CLI and `Goto.app` share `~/.goto` вҖ” plain text, one absolute path per line.
- Finder settings stored in `~/.goto-settings` (JSON).
  - `Goto.app` writes settings, watches the file, and broadcasts to the extension.

## Git branching

- `develop` is the default branch for day-to-day work.
- Open worktrees and PRs from `develop`, not `main`, unless explicitly told otherwise.
- `develop` pushes/PRs should go through the gate runner workflow.
- Merge `develop` into `main` only when you want to cut a deployment/release.

## Architecture rules

1. **Native app consolidation** вҖ” `goto` stays separate, while `Goto.app` owns both menu bar UX and Finder host duties.
2. **Finder Sync extension is sandboxed** вҖ” it cannot read files directly.
   It receives data from `Goto.app` via `DistributedNotificationCenter` IPC.
3. **`Goto.app` is an agent-style app** (`LSUIElement = true`) вҖ” it stays in the menu bar, opens a settings window on demand, and bridges Finder IPC to Terminal.
4. **`Goto.app` watches both `~/.goto` and `~/.goto-settings`** for changes
   and re-broadcasts to the Finder extension automatically.
5. **Terminal auto-detection** uses `NSWorkspace.runningApplications`;
   configurable via `UserDefaults`.
6. **Xcode project is generated** by `scripts/generate_macos_project.rb` вҖ”
   new source files must be registered there.

## Known constraints

- Finder Sync extension menu actions dispatch via XPC вҖ” target must be `self`.
- `directoryURLs` must use the real home path, not `NSHomeDirectory()`
  (which returns the sandbox container path).
- The extension **cannot write** to the filesystem.
- URL scheme `goto-finder://` works from outside the sandbox but **not** from
  the sandboxed extension.
