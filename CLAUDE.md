# goto

> `AGENTS.md` is a symlink to this file.

Three independent packages for macOS project jumping — each installable separately:
**`goto`** (CLI picker), **`goto-menubar`** (SwiftUI menu bar), **`goto-finder`** (Finder Sync headless agent).

## Tech stack

- **CLI** (`goto`): Node.js >= 20, zero dependencies, ESM (`"type": "module"`)
- **Menu Bar** (`goto-menubar`): Swift 5.8, SwiftUI, macOS 13+ — SPM package
- **Finder** (`goto-finder`): Swift 5.8, macOS 13+ — Xcode project (`macos/Goto.xcodeproj`), generated by Ruby script

## Key directories

| Path | Purpose |
|------|---------|
| `src/` | CLI source (commands, select, output) |
| `native/` | SPM package — `GotoNativeCore`, `GotoNativeLaunch`, `GotoMenuBar` |
| `macos/GotoFinder/` | Xcode — headless Finder agent (IPC bridge only) |
| `macos/GotoFinderSync/` | Xcode — Finder Sync extension (sandboxed) |
| `macos/Shared/` | IPC notification definitions shared by agent + extension |
| `shell/` | Shell integration (sourced by user's shell profile) |
| `scripts/` | Build & codegen scripts |
| `test/` | Node.js tests |
| `bin/` | CLI entry point (`goto.js`) |

## Build & test commands

```bash
node --test                          # 24 JS tests — must pass before changes
swift test -c debug                  # 38 Swift tests — must pass before changes
swift build -c release               # Build SPM targets
scripts/build-menu-bar-app.sh        # Build goto-menubar (.app)
scripts/build-finder.sh              # Build goto-finder (.app + .appex)
scripts/install-finder.sh            # Install goto-finder to ~/Applications
scripts/generate_macos_project.rb    # Regenerate Xcode project
```

## Shared registry

- All three packages share `~/.goto` — plain text, one absolute path per line.
- Finder settings stored in `~/.goto-settings` (JSON).
  - `goto-menubar` writes settings, `goto-finder` watches the file and broadcasts to the extension.

## Architecture rules

1. **Three independent packages** — `goto`, `goto-menubar`, `goto-finder` share
   only `~/.goto` and `~/.goto-settings`. No direct IPC between them.
2. **Finder Sync extension is sandboxed** — it cannot read files directly.
   It receives data from `goto-finder` agent via `DistributedNotificationCenter` IPC.
3. **`goto-finder` is a headless background agent** (`LSUIElement = true`) — no
   menu bar icon, no dock icon. It only bridges IPC between the extension and Terminal.
4. **`goto-finder` watches both `~/.goto` and `~/.goto-settings`** for changes
   and re-broadcasts to the Finder extension automatically.
5. **Terminal auto-detection** uses `NSWorkspace.runningApplications`;
   configurable via `UserDefaults`.
6. **Xcode project is generated** by `scripts/generate_macos_project.rb` —
   new source files must be registered there.

## Known constraints

- Finder Sync extension menu actions dispatch via XPC — target must be `self`.
- `directoryURLs` must use the real home path, not `NSHomeDirectory()`
  (which returns the sandbox container path).
- The extension **cannot write** to the filesystem.
- URL scheme `goto-finder://` works from outside the sandbox but **not** from
  the sandboxed extension.

## Maintaining these docs

| When | What to update | How |
|------|---------------|-----|
| CLI 플래그/명령 추가/변경 | README.md "CLI Usage" 섹션 | 새 플래그 + 예제 추가 |

| 아키텍처 결정 변경 | 새 ADR 작성 (`docs/adr/ADR-NNN-*.md`) | 기존 ADR은 수정하지 않고 Status를 Superseded로 변경 |
| 새 제약/함정 발견 | CLAUDE.md "Known constraints" 추가 | 한 줄 요약 + 근거 |
| 테스트 수 변경 | CLAUDE.md "Build & test commands" 주석 업데이트 | `# NN tests` 숫자 갱신 |
| 새 스크립트 추가 | README.md "Scripts" 테이블 | 한 줄 추가 |
| 새 설정 파일 추가 | README.md "Registry" + CLAUDE.md "Shared registry" | 포맷 설명 추가 |

**ADR 작성 규칙:**
- 번호는 순차 증가 (현재 마지막: ADR-006)
- 파일명: `ADR-NNN-kebab-case-title.md`
- 기존 ADR 절대 삭제하지 않음 — Status를 `Superseded by ADR-NNN`으로 변경
- AI에게 "이 결정은 왜 이렇게 했어?"라고 물을 때 ADR이 답이 됨
