# goto

> `AGENTS.md` is a symlink to this file.

Two install surfaces for macOS project jumping:
**`goto`** (CLI picker) and **`Goto.app`** (menu bar UI + Finder Sync host with embedded extension).

## Tech stack

- **CLI** (`goto`): Node.js >= 20, zero dependencies, ESM (`"type": "module"`)
- **Native App** (`Goto.app`): Swift 5.8, SwiftUI, macOS 13+ — Xcode project (`product/macos/Goto.xcodeproj`), generated by Ruby script

## Key directories

| Path | Purpose |
|------|---------|
| `product/cli/src/` | CLI source (commands, select, output) |
| `product/cli/shell/` | Shell integration (sourced by user's shell profile) |
| `product/cli/test/` | Node.js tests |
| `product/cli/bin/` | CLI entry point (`goto.js`) |
| `product/core/` | SPM package — shared native core and `GotoNativeLaunch` |
| `product/macos/Goto/` | Xcode — unified app host (menu bar UI + settings window) |
| `product/macos/FinderBridge/` | Xcode — Finder launch bridge used by `Goto.app` |
| `product/macos/GotoFinderSync/` | Xcode — Finder Sync extension (sandboxed) |
| `product/macos/artwork/` | Source artwork for the native app |
| `product/macos/Resources/` | Native app resources such as `Goto.icns` |
| `product/macos/Shared/` | IPC notification definitions shared by agent + extension |
| `scripts/` | Build & codegen scripts |

## Build & test commands

```bash
node --test product/cli/test/*.test.js           # 27 JS tests — must pass before changes
swift test --package-path product/core           # 32 Swift tests — must pass before changes
swift build --package-path product/core -c release  # Build SPM targets
scripts/build-app.sh                 # Build Goto.app (.app + .appex)
scripts/install-app.sh               # Install Goto.app to ~/Applications
scripts/generate_macos_project.rb    # Regenerate Xcode project
```

## Shared registry

- The CLI and `Goto.app` share `~/.goto` — plain text, one absolute path per line.
- Finder settings stored in `~/.goto-settings` (JSON).
  - `Goto.app` writes settings, watches the file, and broadcasts to the extension.

## Git branching

- `develop` is the default branch for day-to-day work.
- Open worktrees and PRs from `develop`, not `main`, unless explicitly told otherwise.
- `develop` pushes/PRs should go through the gate runner workflow.
- Merge `develop` into `main` only when you want to cut a deployment/release.

## Architecture rules

1. **Native app consolidation** — `goto` stays separate, while `Goto.app` owns both menu bar UX and Finder host duties.
2. **Finder Sync extension is sandboxed** — it cannot read files directly.
   It receives data from `Goto.app` via `DistributedNotificationCenter` IPC.
3. **`Goto.app` is an agent-style app** (`LSUIElement = true`) — it stays in the menu bar, opens a settings window on demand, and bridges Finder IPC to Terminal.
4. **`Goto.app` watches both `~/.goto` and `~/.goto-settings`** for changes
   and re-broadcasts to the Finder extension automatically.
5. **Terminal auto-detection** uses `NSWorkspace.runningApplications`;
   configurable via `UserDefaults`.
6. **Xcode project is generated** by `scripts/generate_macos_project.rb` —
   new source files must be registered there.

## Known constraints

- Finder Sync extension menu actions dispatch via XPC — target must be `self`.
- `directoryURLs` must use the real home path, not `NSHomeDirectory()`
  (which returns the sandbox container path).
- The extension **cannot write** to the filesystem.
- URL scheme `goto-finder://` works from outside the sandbox but **not** from
  the sandboxed extension.

- 번호는 순차 증가 (현재 마지막: ADR-006)
- 파일명: `ADR-NNN-kebab-case-title.md`
